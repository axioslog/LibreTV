# LibreTV v3.2.0 修复验证报告

## 🔧 修复内容

### 修复 1：Service Worker 重定向问题

**问题**：
```
The FetchEvent for "https://libretv-6gq.pages.dev/watch.html?..." resulted in a network error response: a redirected response was used for a request whose redirect mode is not "follow".
```

**根本原因**：
- Service Worker 拦截了 `watch.html` 的重定向请求
- 没有正确设置重定向模式
- 导致页面无法正常跳转到播放器

**解决方案**：
- 为 `watch.html` 和 `player.html` 添加特殊处理
- 不使用缓存，直接请求并设置 `redirect: 'follow'`
- 确保重定向请求能正常处理

**影响文件**：
- `service-worker.js`

#### 第二步：清除浏览器缓存（重要！）

**必须清除以下所有缓存**：

1. **Service Worker**：
   - 按 `F12` 打开开发者工具
   - 进入 "Application" > "Service Workers"
   - 点击 "Unregister" 注销旧 Service Worker

2. **IndexedDB 数据**：
   - 在开发者工具中进入 "Application" > "Storage" > "IndexedDB"
   - 找到 "LibreTVOffline" 数据库
   - 右键点击并选择 "Delete database"

3. **浏览器缓存**：
   - 按 `Ctrl + Shift + R` 强制刷新
