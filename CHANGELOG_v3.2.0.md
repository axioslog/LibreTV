# LibreTV v3.2.0 更新日志

## 发布日期
2026-04-24

## 更新内容

### 🎉 新增功能

#### 1. 高效下载管理器
- **DownloadManager**: 全新的下载管理器，支持断点续传、多线程下载
- **M3U8Downloader**: 专门的M3U8下载器，支持分片下载和合并
- **DownloadProgressManager**: 下载进度管理器，实时跟踪下载状态

#### 2. 下载优化
- **断点续传**: 支持从中断处继续下载，避免重复下载
- **多线程下载**: 支持并发下载多个视频片段，提高下载速度
- **进度管理**: 实时显示下载进度和速度
- **错误重试**: 自动重试失败的下载任务，最多重试5次
- **超时控制**: 添加30秒超时机制，避免长时间卡住

#### 3. M3U8解析优化
- **超时机制**: M3U8解析添加30秒超时，避免长时间卡在"解析中"状态
- **错误处理**: 改进错误处理，提供更友好的错误提示
- **重试机制**: 分片下载失败时自动重试，最多重试5次

### 🐛 修复问题

#### 1. 缓存管理页面无法打开
- **问题**: offline-cache-enhanced.js缺少必要的IndexedDB辅助函数
- **修复**: 添加了`getAllOfflineVideos`、`getOfflineVideo`、`saveOfflineVideo`、`deleteOfflineVideo`等函数
- **影响**: 缓存管理页面现在可以正常打开和使用了

#### 2. "解析中"状态卡住
- **问题**: M3U8解析没有超时机制，网络问题时会长时间卡住
- **修复**: 添加30秒超时机制，超时后显示友好的错误提示
- **影响**: 下载失败时会及时提示用户，不再长时间卡住

#### 3. 分片下载失败
- **问题**: 分片下载失败时没有重试机制
- **修复**: 添加自动重试机制，最多重试5次，使用指数退避策略
- **影响**: 提高了下载成功率，减少了因网络波动导致的下载失败

### 🔧 技术改进

#### 1. 代码结构优化
- 新增`js/download-manager.js`文件，包含所有下载相关功能
- 优化`js/offline-cache-enhanced.js`，添加缺失的IndexedDB辅助函数
- 改进`js/player.js`中的下载逻辑，添加超时和重试机制

#### 2. 性能优化
- 使用指数退避策略进行重试，避免频繁重试导致服务器压力
- 优化并发控制，最多同时下载6个分片
- 添加下载速度监控，实时显示下载速度

#### 3. 用户体验优化
- 改进错误提示，提供更友好的错误信息
- 添加下载进度实时显示
- 优化缓存管理界面，添加存储空间管理功能

### 📝 文件变更

### 🚀 部署说明

1. **本地测试**
   ```bash
   npm run dev
   ```

2. **构建部署**
   ```bash
   npm run build
   ```

3. **Cloudflare Pages部署**
   - 推送代码到GitHub
   - Cloudflare Pages会自动构建和部署

### ⚠️ 注意事项

1. **Service Worker更新**
   - 由于更新了Service Worker的缓存版本，用户需要刷新页面才能获取最新版本
   - 建议在更新后清除旧缓存

2. **IndexedDB版本**
   - IndexedDB版本从5升级到6
   - 旧数据会自动迁移，但建议用户清理旧缓存

3. **浏览器兼容性**
   - 需要支持ES6+的现代浏览器
   - 需要支持IndexedDB、Service Worker、Fetch API
